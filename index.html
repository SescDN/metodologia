UX

O que é UX?

UX é uma abreviação do termo inglês User Experience, que significa Experiência do Usuário, ou seja, é o que o usuário sente ao utilizar qualquer produto ou serviço.

...

Você lembra da última vez que comprou um celular novo?

Quando começou a utilizar, se sentiu feliz com a leveza e brilho do aparelho e adorou a rapidez do sistema. Por outro lado, o seu celular antigo travava com frequência, descarregava rápido e tinha mau contato nos botões, uma experiência nada agradável.

UX é o termo que usamos para avaliar exatamente isso.

Podemos perceber que cada detalhe, desde os ícones dos aplicativos até o encaixe do aparelho na sua mão, foi minuciosamente planejado. Essa responsabilidade cabe ao UX Designer, que precisa reunir diversas habilidades em seu perfil, principalmente o que chamamos de empatia.

Empatia

Em poucas palavras, empatia é se colocar no lugar do outro, buscando pensar e agir como ele, compreendendo suas reais necessidades.

Ser empático é responder a pergunta "O que o meu usuário faria nessa situação?".

...

Como UX Designer, é seu trabalho criar uma experiência útil, acessível e prazeirosa. Um usuário feliz ama seu produto/serviço e se torna um divulgador da sua marca!

Mas, para que isso seja realmente possível, é preciso primeiro conhecê-lo.

"O que o meu usuário faria nessa situação?"

Prazer, usuário

Conhecer o público alvo do seu produto ou serviço é como ficar íntimo de qualquer pessoa, é preciso tempo e dedicação.

Design Thinking

Um das ferramentas essenciais para o pensamento de Design é o método Duplo Diamante, que acompanha o designer até a Prototipação, o principal entregável de todo projeto de UX.

...

Pesquisa (por quê?)
Todo projeto nasce da percepção de um problema. Essa etapa funciona como um diagnóstico, para que os envolvidos possam entender as condições atuais do projeto e por que estamos fazendo isso.

Descoberta (quem?)
Esse momento é utilizado para que a equipe envolvida possa compreender seus usuários e descobrir oportunidades.

Ideação (como?)
Como o nome diz, é a parte criativa do projeto. Aqui as perguntas geradas no primeiro diamante começam a ser respondidas.

Prototipação (como?)
Fase onde as ideias tomam forma, sendo testadas e refinadas para, enfim, se tornarem uma solução real.

Prototipação

O que é um protótipo?

É uma versão simulada do produto final, utilizada para testes de layout, usabilidade e identidade visual antes do seu desenvolvimento.

O mais importante é sempre manter seus usuários e stakeholders próximos durante o processo, eles vão ajudá-lo a construir uma solução que realmente resolva um problema.

...

Para desenvovler um bom protótipo é preciso primeiro saber a quem ele se destina:

Stakeholders
Protótipo de baixa fidelidade para validar se o produto/serviço é realmente aquilo que ele tinha em mente.

Desenvolvedores
Protótipo de média fidelidade para guiar o desenvolvedor em sua implementação.

Usuários
Protótipo de alta fidelidade para buscar feedbacks de pessoas reais e melhorar o produto em si.

Como tomar decisões corretas?

A maioria das ferramentas de prototipagem possuem recursos que auxiliam a construção de interfaces seguindo as boas práticas de UX, mas é importante que o designer entenda bem os conceitos essenciais para não perder tempo com o que já sabemos que não funciona.

Felizmente, existem as 8 regras de ouro do Design de Interfaces que podem ajudar você a identificar a decisão correta a ser tomada em cada situação.

...

Você sempre pode fazer testes de usabilidade, Analytics, testes A/B e outros para testar suas interfaces, mas isso tem um custo de tempo e orçamento. Um caminho mais fácil é seguir regras já testadas e aprovadas que tornarão seu trabalho muito mais eficiente.

Uma boa prática é nunca começar seu trabalho com uma folha ou tela em branco. Pesquise referências, veja o que funciona ou não em interfaces semelhantes, crie uma vasta biblioteca visual, deixe sua mente trabalhar no problema e só então pegue no lápis ou no mouse.

"O maior desafio do UX Designer é não reinventar a roda."

Web Design

O que é web design?

O Web Design é uma área com foco no desenvolvimento de interfaces de usuário, seus componentes e aplicações para a web. É responsável por “traduzir” para código toda a interface desenhada pelo profissional de UX.

É a tradução da harmonia entre beleza e funcionalidade contida no protótipo para código. Afinal, nem tudo que funciona agrada o olhar, e nem tudo que é bonito é útil.

Qual é a vantagem do Web Design?

Garantir que elementos da interface e comportamentos definidos no protótipo sejam respeitados.

Isto compreende: espaçamentos, tipografia, cores, ergonomia, arquitetura da informação, navegabilidade e outras características mais próprias da interface, como sombras, botões, tabelas, etc.

...

O projeto deve ter fidelidade ao design que foi aprovado anteriormente. Ter estes elementos corretamente posicionados trará profissionalismo ao projeto, além de reforçar o propósito da instituição e sua identidade visual.

Conceitos de Web Design

Por que um web designer e não um desenvolvedor?

Desenvolvedores são mais focados na lógica de programação e na comunicação entre servidor e interface, do que em layout, especificamente.

Questões de design não são funções da área de desenvolvimento. A etapa do web design existe para que tenhamos sempre e depois um visual consistente, de acordo com o protótipo e com a intenção da marca.

Certo, mas qual é a diferença no meu projeto?

Um web designer traz a ideia de um protótipo, de forma visual, para a realidade e que experiência de uso a aplicação deve conter.

O desenvolvedor web cria a estrutura principal do site de forma dinâmica, conectando o visual com dados.

...

Assim como mostarda e ketchup em um cachorro-quente, você precisa de um UX, web designers e desenvolvedores para obter a melhor experiência. Ao criar uma aplicação, você precisa de um desenvolvedor web para construir a base e de web designers para fazer com que pareça e sinta da maneira que foi aprovado no protótipo, construído pelo UX Designer.

Design System

Desenvolvimento de frontend

Com os componentes já desenvolvidos, as próximas interfaces que sigam o mesmo padrão visual, podem ter os mesmos componentes.

Em geral, na primeira interface a ser desenvolvida o tempo e as dúvidas costumam ser bem maiores. Após estas etapas, em que os componentes já existem, esse tempo também cai, desde que não haja mudanças estruturais no projeto.

Open Source

Projeto open source

O Design System do Sesc é um projeto open source, o que significa que código fonte é disponibilizado com uma licença de código aberto. Fornece o direito de utilizar, modificar e distribuir o projeto sem fins lucrativos.

Todos os componentes para desenvolvimento de interfaces de Front-End estão disponíveis para uso livre, além de um StarterKit totalmente otimizado com templates de interfaces prontos de uso cotidiano.

O StarterKit

O StarterKit é um conjunto de interfaces prontas, com dados mocados, que tem o intuito de agilizar a criação de sistemas ou aplicações que utilizem uma mesma estrutura de colunas, tabelas e componentes. Desta forma, o trabalho do desenvolvedor é reduzido a alterar os dados para o objetivo do sistema e fazer as conexões com o Back-End.

Já o Design System fornece os componentes prontos, para que a construção seja mais rápida e livre de erros e bugs.

Github

Ao hospedarmos o Design System com os componentes em um projeto open source através do Github, temos as seguintes vantagens:

É possível planejar e desenvolver projetos inovadores e incluir participantes.

Compartilhar projetos como o Design System e o StarterKit, possibilitando a implementação de um pacote de componentes e interfaces visuais prontas para serem integradas

Fazer sugestões de melhoria em componentes e no projeto como um todo

Como fazer parte

Todo o conteúdo do Design System e do StarterKit é open source. Todos todos terão acesso e poderão clonar o projeto, além de contribuir e sugerir melhorias.

O Design System é um produto que sempre terá novos componentes sendo adicionados, facilitando o desenvolvimento seguindo a identidade e padrões visuais da instuição.
